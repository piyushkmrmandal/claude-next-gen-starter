# UI PATTERNS

## Approved Patterns

### Hero Sections
- Full-viewport with asymmetric text placement
- Typography-led with strong visual anchor
- Subtle background texture or noise (not gradients)
- Single, weighted CTA

### Navigation
- Sticky header with blur backdrop
- Logo left, links center or right
- No hamburger menus on desktop
- Mobile: drawer or bottom navigation

### Cards
- 24px border radius
- Subtle border (1px, low opacity)
- Hover: lift with shadow + border brightening
- No header/body/footer card structure — use free layout

### Buttons
- Primary: solid, high contrast, pill or slightly rounded
- Secondary: outline or ghost, matching border-radius
- Destructive: never red on dark — use muted warning tone
- Loading: spinner replaces icon, not text

### Forms
- Floating labels or always-visible labels (no placeholder-only)
- Error states: inline, below field, red dot indicator
- Success: green check, not disruptive
- Grouped with 16px gap between fields

### Empty States
- Centered illustration or icon
- Helpful heading (what to do, not what's missing)
- Single action CTA
- No "No data found" text

## Patterns to Avoid
- Accordion menus as primary navigation
- Tooltip on every icon
- Toast notifications for every action
- Modal confirmation for low-risk actions
